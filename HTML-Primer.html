The HTML Primer

Learning to work with HTML in Learning Management Systems (LMS) helps improve student user experience by:

  • ensuring semantic content presentation,
  • compliment organizational brand adherence,
  • allow for modularity/reusability of curriculum materials,
  • improving accessibility and User Experience (UX) for all students.

The HTML Primer code was developed as a resource for web designers/developers, educators, and support staff working with any LMS tool to:

  • easily understand core HTML code,
  • visualize predefined styling of content within the LMS,
  • comprehend how HTML elements available can be applied to eLearning curriculum,
  • encourage an HTML first philosophy.

This HTML Primer is intended to be used as a testing mechanism of an institutions LMS design and capabilities, an educational resource for educators and support staff, and a collection of repeatable HTML code (yes, you will need to look at the HTML) that can be implemented into any webpage within a LMS or CMS.

HOW TO USE

In any LMS or CMS, simply copy all the HTML code in the HTML Primer document and paste this code into a new page within your organizations LMS using the HTML editing function.

Educators & Support Staff

The HTML Primer resource is intended to be used by eLearning educators and support staff to develop an understanding and appreciation of HTML code that is specifically content focused. By building an appreciation for an HTML first strategy to deliver curriculum in any LMS will ensure students receive an eLearning experience that is accessible, user centric, semantic, and compatible with the institution's intended LMS design.

Web Developers & Developers

Web designers and developers can leverage this HTML Primer to determine gaps in the LMS or CMS content design. This code aids skilled designers/developers to customize the look and feel of the LMS content (through custom CSS rules) to compliment the organization's brand, ensure best practices are followed for accessibility, and to leverage semantic HTML that will compliment effective user experience (UX). By ensuring the content is presented correctly, this empowers the instructor and/or support staff to integrate curriculum with little to none design customizations to the material within their course hosted on the LMS.

Headings

The following is HTML heading elements <H1> through <H6>. Each heading represents descending order of importance which can be indicated also by font, size, colour, and more depending on styling rules.

<H1> Heading

Considered to be the main heading on any webpage. This element should only be used once (per page) as the rest of the page content should be sub to the <h1>. Typically, the <h1> will be already used by the LMS as it will be the main header for any page in your eLearning experience.

<H2> Heading

Both the <h2> and <h3> headings are the most commonly used when defining hierarchy in your content. By using descending headings, you are creating the necessary semantic structure (Kyrnin, 2020) impacts how various accessibility tools interpret the material and overall perception of the learning experience.

<H3> Heading

In the Learning Management System (LMS), the look and feel is pre-determined by the organization/company and modified by institutions developers to compliment branding and colour schemes (as outlined by brand guidelines).

<H4> Heading

Aside from branding and colour rules, other elements of the webpage (such as paragraphs <p>, links <a>, lists <ul> or <ol>, tables <table>, and more) build on the semantic structure and adds to the context of the curriculum.

<H5> Heading

Browsers by default have a common set of design rules on how elements will look when a webpage is loaded. Web designers/developers customize the web browsing experience by establishing a custom set of rules by programming Cascading Style Sheets (CSS) to compliment the organizational branding guidelines.

<H6> Heading

Coding HTML requires the opening and closing of element code. Creating a heading in HTML is achieved by wrapping your text with the heading element code: <h6>Heading</h6>.

Learn more about headings at Mozilla.org.


Paragraphs

A majority of your content will be presented using the <p> and can work with other elements such as lists, links, tables, blockquotes and more. Presenting your content in paragraphs is important as it ensures semantic structure, effective styling (fonts, colours, size, line height, etc) and consistency. Review this example of paragraphs in use:

Lists

Lists are a great way to break up content in to easy to read semantic elements. There are three different types of lists used in HTML:

  • unordered <ul>
  • ordered <ol>
  • description <dl>

Ordered and Unordered lists are the most common and are able to be embedded within each other to create deeper contextual value.

UNORDERED LISTS

Unordered lists use bullets to illustrate content in logical point form. By default unordered lists start with the solid circle (•), however disk (â—‹) and square (■) are also options that are used depending on the lists embedding depth or by adding in-line styles rules to the unordered list element <ul>. Review this example of unordered lists in use:

Unordered lists example
HTML: Result:
<ul>
 <li>Quiz</li>
 <li>Midterm</li>
 <li>Final</li>
</ul>
  • Quiz
  • Midterm
  • Final

Learn more about unordered lists at Mozilla.org.

Ordered lists use numerical, alpha, or roman numerals to illustrate a sequential order of information. By default ordered lists will be numbered 1, 2, 3, 4, etc. Review this example of ordered lists in use:

 <li>Quiz</li>
 <li>Midterm</li>

EXTERNAL vs. INTERNAL

When using hyperlinks, one of the most common linking options is to consider whether or not to open the link in a new window. Common rule of thumb; when hyperlinking to a webpage external to your website or linking to a document, set up the link to open in a new window or tab. By including target="_blank" to the anchor, the link will open in a new window/tab.

EXTERNAL vs. INTERNAL Hperlinks example
HTML: Result:

 https://developer.mozilla.org/en-US/
</a> 		https://developer.mozilla.org/en-US/

Tables

The use of tables in HTML are an excellent option to organize data, layout a schedule, or to highlight contextual relationships. The <table> can be sectioned into the head (<thead>), body (<tbody>) and if needed the footer (<tfoot>). Table rows are indicated by the <tr> element and table cells are <td> (unless in <thead>, then use <th>).

Tables example
HTML: Result:
<table>
 <caption>Number of Students in Classes</caption>  <thead>
  <tr>
   <th scope="col">Class:</th>
   <th scope="col">Students:</th>
  </tr>
 </thead>
 <tbody>
  <tr>
   <td>MATH 10</td>
   <td>50</td>
  </tr>
  <tr>
   <td>SOCIAL 10</td>
   <td>45</td>
  </tr>
  <tr>
   <td>SCIENCE 10</td>
   <td>55</td>
  </tr>
  </tbody>
 <tfoot>
  <tr>
   <th scope="row">Total Students:</th>
   <td>150*</td>
  </tr>
 </tfoot>
</table>

Contextual Elements

These various contextual elements can help elevate your content and provide additional semantic context for the user. These serve to enhance the text contextual value for improved user experience and accessibility.

STRONG

Strong example

QUOTES

In HTML, there are two types of quote elements (<q> & <blockquote>) to indicate that the inserted text is representative of a speakers perspective.

Quote

Quote (<q>) element is used for small quotes within a sentence. Embedding a quote within each other will change the quote characteristic from a double quote to a single quote. Learn more about quote at Mozilla.org.

Quote example
HTML: Result:
Bobbi told me, <q>Delia said, <q>This will never work.</q></q> Bobbi told me, Delia said, This will never work.

The <q> ensures that the text wrapped with this element carries the semantic meaning of being quoted. It is possible to manually quote text by just using the quotation marks as browsers render quotes mark by default in the simple format (" "). If you are looking to manually include non-semantic traditional open and close quotes without using <q>, use the following HTML entities:

  • Double quotes:
    • Open (&ldquo;): “
    • Close (&rdquo;): ”
  • Single quotes:
    • Open (&lsquo;): ‘
    • Close (&rsquo;): ’

Blockquote

Blockquote (<blockquote>) element is used for quotes that are the entire sentence or paragraph. Using blockquote is ideal to add pull quotes into your curriculum to create visual interest and to make the quote stand out. Learn more about blockquote at Mozilla.org.

Line Break

The <br> is used to create a line break in text within a paragraph. It is useful for writing poems or addresses when separate lines are required within a paragraph. A common mistake occurs in the LMS or CMS editor when users press the ENTER key twice to give the visual impression of paragraphs; but in fact the text blocks are separated by multiple <br> elements to create the impression of paragraphs instead of correctly wrapping the content with <p>. Learn more about line break at Mozilla.org.

<p>Then took the other, as just as fair,<br>

The Road Not Taken

By Robert Frost

Two roads diverged in a yellow wood,
And sorry I could not travel both
And be one traveler, long I stood
And looked down one as far as I could
To where it bent in the undergrowth;

Then took the other, as just as fair,
And having perhaps the better claim,
Because it was grassy and wanted wear;
Though as for that the passing there
Had worn them really about the same,

And both that morning equally lay
In leaves no step had trodden black.
Oh, I kept the first for another day!
Yet knowing how way leads on to way,
I doubted if I should ever come back.

I shall be telling this with a sigh
Somewhere ages and ages hence:
Two roads diverged in a wood, and I–
I took the one less traveled by,
And that has made all the difference.
